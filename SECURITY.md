# Security Policy

## Supported Versions

Since this is a personal portfolio website (not a production library or software package), security updates are handled as follows:

| Version       | Supported          |
|---------------|--------------------|
| Latest        | :white_check_mark: |
| Older versions| :x:                |

Only the latest version of the website is actively maintained.

## Reporting a Vulnerability

If you discover a security vulnerability in this project (for example: XSS, broken links, or any other issue), please report it responsibly.

### How to Report:
- **Preferred**: Open a new issue on GitHub with the title **"[Security] Vulnerability Report"**
- **Alternative**: You can also email me directly at: `sourya791m@gmail.com` (if you prefer private reporting)

### What to Expect:
- I will acknowledge your report within **48 hours**.
- I will investigate and provide an update within **7 days**.
- If the vulnerability is accepted, I will work on fixing it as soon as possible and credit you (if you wish).
- If declined, I will explain the reason clearly.

Your help in making this project safer is greatly appreciated!

---

**Note**: This is a personal portfolio website built for learning and showcasing projects. It does not handle sensitive user data or critical systems.
